# TR&TS (Talent Relationship & Tracking System) — PRD

## 1. 서비스 개요
TR&TS는 리크루터를 위한 사내 소싱·파이프라인 관리 도구입니다.
JD 기반 소싱 메시지 자동 생성, 후보자별 개인화, 이력서 DB화,
채용 현황 트래킹을 한 곳에서 처리합니다.
참고 서비스: 나인하이어, 그리팅, Greenhouse

## 2. 폴더 구조
이 레포는 5개 팀(파트)에 따른 5개 영역으로 나뉜다:
- planning/ — 기획 (무엇을 만들 것인가)
- design/ — 디자인 (어떻게 보여줄 것인가)
- cx/ — CX (누가 어떻게 사용할 것인가)
- marketing/ — 마케팅 (어떻게 알릴 것인가)
- strategy/ — 전략 (어떻게 지속 가능하게 만들 것인가)

각 영역의 상세는 해당 폴더에서 작성된다.

## 3. 기획 (Planning)
- 소싱 메시지 고도화: JD 입력 → 포지션 맞춤 템플릿 생성·저장
- 소싱 메시지 커스텀: 템플릿 + 후보자 이력 → 개인화 메시지
- 인재 DB: 이력서 PDF 또는 이미지 파일(JPG·PNG) 업로드 → 자동 파싱 → Notion DB 저장
- 대시보드: 채용 파이프라인 인원 트래킹·관리
MVP는 위 4개 기능 전체.

상세는 `planning/` 폴더 참조. [작성 예정]

## 4. 디자인 (Design)
깔끔하고 효율적인 B2B 내부 도구 스타일.
메인 대시보드 → 후보자 상세 → 소싱 메시지 편집 흐름.

상세는 `design/` 폴더 참조. [작성 예정]

## 5. CX (Customer Experience)
주 사용자는 리크루터. 반복 작업(메시지 작성, DB 입력)을 자동화해 시간 절약.
인지 → 온보딩 → 일상 사용 → 팀 내 확산.

상세는 `cx/` 폴더 참조. [작성 예정]

## 6. 마케팅 (Marketing)
사내 도구이므로 내부 채용팀 온보딩이 핵심.
성공 지표: 이용자 수, 월간 활성 사용자(MAU).

상세는 `marketing/` 폴더 참조. [작성 예정]

## 7. 전략 (Strategy)
무료 사내 도구. 활성화율과 리크루터 만족도가 지속 가능성 지표.
추후 Notion·Slack·Google Workspace 연동으로 확장.

상세는 `strategy/` 폴더 참조. [작성 예정]

## 8. 역할 배정 (가상)
이 프로젝트를 팀으로 한다면:

| 팀 | 담당 영역 | 작업 폴더 |
|----|---------|----------|
| PM (본인) | 전체 기획·의사결정 | 루트 |
| 기획 팀 | 기능 정의, MVP 범위 | planning/ |
| 디자인 팀 | UI/UX, 비주얼 | design/ |
| CX 팀 | 페르소나, 여정맵, FAQ | cx/ |
| 마케팅 팀 | 런칭, 채널 | marketing/ |
| 전략 팀 | 수익 모델, 성장 | strategy/ |
