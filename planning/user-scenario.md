# 리크루터 사용 시나리오

TR&TS를 처음 접속한 리크루터가 소싱 메시지를 발송하기까지의 전체 흐름.

---

## 페르소나

- **이름**: 김민지 (가상)
- **직책**: IT 기업 채용 담당자
- **목표**: 백엔드 개발자 3명 채용 / 이번 달 안에 소싱 완료
- **페인포인트**: 포지션마다 소싱 메시지를 처음부터 쓰는 게 너무 시간이 걸림

---

## 시나리오 A — 처음 쓰는 날 (소싱 메시지 자동생성)

```
1. TR&TS 접속

2. "소싱 메시지 생성" 진입

3. JD 입력
   └── 회사 채용 공고 URL 붙여넣기 → 자동 파싱 확인

4. 버전 설정
   ├── 버전 1: "Java/Spring 경력 강조"
   └── 버전 2: "스타트업 문화 어필"

5. 추가 지시사항 입력 (선택)
   └── "친근하고 짧게, 3줄 이내로"

6. 생성 → 2개 버전 메시지 확인

7. 버전 1 선택 후 일부 수정

8. 저장 → "백엔드 개발자 (5년+)" 포지션으로 보관
```

---

## 시나리오 B — 후보자별 개인화

```
1. "소싱 메시지 개인화" 진입

2. 저장된 템플릿 선택
   └── "백엔드 개발자 (5년+)" 선택

3. 후보자 정보 입력
   └── 링크드인 URL 붙여넣기 → 파싱 완료

4. 개인화 생성

5. 결과 확인
   └── "Java Spring 10년 경험을 보유하신 홍길동 님께..."

6. 복사 → 링크드인 메시지로 전송
```

---

## 시나리오 C — 이력서 DB 저장

```
1. 지원자 이력서 PDF 수신

2. "인재 DB" 진입 → 파일 업로드

3. 파싱 결과 확인 및 수정

4. Notion DB 저장 완료
```

---

## 기능 간 연결 흐름

```
[소싱 메시지 생성] ──저장──▶ [템플릿 라이브러리]
                                      │
                                      ▼
[인재 DB] ────후보자 선택────▶ [소싱 메시지 개인화]
                                      │
                                      ▼
                              [개인화 메시지 발송]
                                      │
                                      ▼
                            [채용 파이프라인 대시보드]
```
