# Feature 01 — 소싱 메시지 자동생성

> 상태: 명세 작성 중

## 기능 개요

JD(채용공고)를 입력하면 리크루터가 지정한 버전 수만큼 소싱 메시지를 자동 생성한다.

---

## 유저 스토리

> "리크루터로서 나는, JD를 입력하고 원하는 버전 방향을 지정하면  
> 각 버전에 맞는 소싱 메시지를 즉시 받고 싶다.  
> 그래야 매번 메시지를 처음부터 작성하는 시간을 줄일 수 있다."

---

## 입력

| 항목 | 필수 여부 | 설명 |
|------|---------|------|
| JD 입력 | 필수 | URL 자동 파싱 또는 텍스트 직접 붙여넣기 |
| 추가 지시사항 | 선택 | 리크루터가 자유롭게 입력하는 방향성 (예: "친근한 톤으로", "기술 스택 강조") |
| 버전 레이블 | 최소 1개 | 생성할 버전의 이름/방향 입력 (예: "SAP 특화", "연결회계 특화") |

---

## 출력

- 버전 레이블 수만큼 개별 소싱 메시지 생성
- 각 메시지는 편집 가능
- 저장 시 템플릿 라이브러리에 포지션명으로 보관

---

## 상세 플로우

```
1. JD 입력
   ├── URL 입력 → 자동 파싱 (포지션명, 주요 업무, 자격요건 추출)
   └── 텍스트 붙여넣기 → 그대로 사용

2. 추가 지시사항 입력 (선택)
   └── 자유 텍스트 입력창

3. 버전 레이블 입력
   ├── 기본 1개 입력창 제공
   └── "+ 버전 추가" 버튼으로 최대 N개까지 추가

4. "생성" 버튼 클릭
   └── 각 버전 레이블 + JD + 지시사항 → AI 메시지 생성

5. 결과 확인
   ├── 버전별 탭 또는 카드 형태로 나란히 표시
   └── 각 메시지 직접 편집 가능

6. 저장
   └── 포지션명 태그와 함께 템플릿 라이브러리에 저장
```

---

## 엣지 케이스

| 상황 | 처리 방식 |
|------|---------|
| URL 파싱 실패 (접근 불가, 로그인 필요 등) | 오류 메시지 + 텍스트 붙여넣기로 전환 안내 |
| 버전 레이블 미입력 | "기본" 레이블로 1개 생성 |
| JD 내용 없이 생성 요청 | JD 입력 필드 필수 유효성 검사 |
| 생성 중 네트워크 오류 | 재시도 버튼 제공, 입력값 유지 |

---

## 연관 기능

- **Feature 02 (개인화)**: 이 기능에서 저장된 템플릿을 불러와 개인화에 사용
- **Feature 03 (인재 DB)**: 후보자 DB와 연결해 개인화 시 후보자 정보 자동 불러오기
