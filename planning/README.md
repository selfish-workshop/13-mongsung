# 기획 (Planning)

이 폴더는 TR&TS의 **"무엇을 만들 것인가"** 상세 명세를 담습니다.
개발팀이 기능 단위로 참조할 수 있도록 기능별 파일로 분리합니다.

---

## MVP 범위

총 4개 핵심 기능을 MVP로 정의합니다.

| # | 기능 | 파일 | 상태 |
|---|------|------|------|
| 1 | 소싱 메시지 자동생성 | `feature-01-sourcing-template.md` | 명세 작성 중 |
| 2 | 소싱 메시지 개인화 | `feature-02-personalization.md` | 명세 작성 중 |
| 3 | 인재 DB | `feature-03-talent-db.md` | 명세 작성 중 |
| 4 | 채용 파이프라인 대시보드 | `feature-04-dashboard.md` | 명세 작성 중 |

---

## 우선순위 및 개발 순서

리크루터가 가장 먼저 가치를 느끼는 순서 기준:

```
1순위  소싱 메시지 자동생성   → 핵심 가치, 즉시 생산성 향상
2순위  소싱 메시지 개인화     → 1순위와 연결, 퀄리티 향상
3순위  인재 DB               → 후보자 자산 축적
4순위  채용 파이프라인 대시보드 → 현황 가시성 확보
```

---

## 3주 MVP 로드맵

| 주차 | 개발 범위 |
|------|---------|
| 1주차 | 소싱 메시지 자동생성 (JD 입력 → 버전별 메시지 생성 → 저장) |
| 2주차 | 소싱 메시지 개인화 (후보자 정보 입력 → 개인화 메시지 생성) |
| 3주차 | 인재 DB + 채용 파이프라인 대시보드 |

## 문서 작성 규칙

각 `feature-*.md` 파일은 아래 항목을 포함합니다:
- **기능 개요** — 한 줄 정의
- **유저 스토리** — "리크루터로서 나는 ~을 원한다"
- **입력 / 출력** — 무엇을 받고 무엇을 반환하는가
- **상세 플로우** — 단계별 동작
- **엣지 케이스** — 예외 상황 처리
- **연관 기능** — 다른 기능과의 연결점
